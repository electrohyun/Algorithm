## 씨름했던 지점

| 문제        | 함정                                 | 되짚을 포인트                                |
| ----------- | ------------------------------------ | -------------------------------------------- |
| K번째수     | `for...in` 은 인덱스("0","1")를 준다 | 배열 원소 순회는 `for...of`                  |
| K번째수     | 자른 뒤 **정렬**을 빼먹음            | "자르고 → 정렬 → k번째" 순서                 |
| K번째수     | 1-based ↔ 0-based / slice 끝 미포함  | 시작만 `-1`, 끝(`j`)은 그대로, k번째는 `k-1` |
| 문자열 정렬 | `a[n] - b[n]` → 문자 뺄셈은 `NaN`    | 문자/문자열은 부등호(`<` `>`)로 사전순 비교  |
| 문자열 정렬 | comparator 가 불리언 반환            | comparator 는 **숫자** -1/0/1 을 반환해야 함 |

## 문제

```js
// ── p1) 문자열 뒤집기 ── "jaron" -> "noraj"
function p1(my_string) {
  return my_string.split("").reverse().join("");
}

// ── p2) K번째수 ──
function p2(array, commands) {
  const answer = [];

  for (const [firstIndex, lastIndex, targetIndex] of commands) {
    answer.push(
      array.slice(firstIndex - 1, lastIndex).sort((a, b) => a - b)[
        targetIndex - 1
      ],
    );
  }

  return answer;
}

// ── p3) 문자열 내 마음대로 정렬하기 ──
function p3(strings, n) {
  return strings.sort((a, b) => {
    if (a[n] < b[n]) return -1;
    if (a[n] > b[n]) return 1;
    if (a < b) return -1;
    if (a > b) return 1;
    return 0;
  });
}
```

- 2번 문제의 경우 a[1] - 1처럼 생각이 닿는대로 풀었는데, const lastIndex = ... 이런 식으로 푸는 연습을 반드시 하자. 내가 그동안 속도와 정답에만 집중해왔더라도, 이런 식의 분리는 손해 볼 게 없다고 생각한다. 오히려 좋다고 생각한다.
