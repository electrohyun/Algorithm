## 문제 요약

- 유저 피로도 `k`, 던전 목록 `dungeons` (각 원소 `[최소 필요 피로도, 소모 피로도]`).
- **현재 피로도 ≥ 최소 필요 피로도**여야 그 던전 입장 가능 → 입장하면 소모 피로도만큼 깎임.
- 던전 순서를 **자유롭게** 정해서 **최대 몇 개**를 돌 수 있는지 반환.
- 예: `k=80`, `[[80,20],[50,40],[30,10]]`
  - 80→(0번,80필요)→60→(1번,50필요)→20→(2번,30필요? 20<30 불가)... 순서 바꿔가며 최댓값 = **3**

## 오늘의 목표 (JS로 굳힐 것)

- 소수찾기의 **순열 나열 골격** 재사용 (던전 순서 = 순열)
- 담는 게 아니라 **최댓값 갱신** — `answer = Math.max(answer, count)`
- **표시(used) → 재귀 → 해제** 백트래킹 한 번 더 손에 붙이기
- 입장 **가능 조건 가지치기**(`fatigue >= 최소 필요`)로 안 되는 가지 잘라내기

## 문제

```js
// 재귀·백트래킹 — 피로도 (87946)
// 던전 순서를 순열로 나열, 입장 가능하면 뻗고, 깬 개수의 최댓값을 남긴다.
function solution(k, dungeons) {
  let answer = 0;

  function game(curK, curDungeons, clearCount) {
    answer = Math.max(answer, clearCount); // 들어올 때마다 최댓값 갱신

    for (const [i, [min, con]] of curDungeons.entries()) {
      if (curK < min) continue;                                   // 피로도 부족 → 건너뜀
      const dungeon = curDungeons.filter((item, idx) => idx !== i); // i번째 뺀 나머지
      game(curK - con, dungeon, clearCount + 1);                  // 깎고 +1 해서 재귀
    }
  }

  game(k, dungeons, 0);
  return answer;
}
```

## 회고

오늘의 발견:

- 재귀 문제의 패턴(이라고 하기엔 2개밖에 안 풀었지만)

잘한 점:

- 오랜만에 빠르게 풀어냈다(자신감 회복)

불확실한 점:

- `for (const [i, [min, con]] of curDungeons.entries())` 로 시작해서,
- `const dungeon = curDungeons.filter((item, idx) => idx !== i);` 처럼 추출하는 테크닉은 반드시 알아둬야 한다고 느낀다.
- 이걸 떠올려내는 것이 중요할 것이다.
