# 이 레포지토리는

알고리즘 공부하는 레포지토리이다.

# 뭘 남기나요

1. 이 알고리즘이 무슨 알고리즘인지(00_about.js)
2. 이 알고리즘이 무슨 알고리즘인지 설명(00_about.md)
3. 시각화해서 살펴보기(00_playground.html)
4. 문제 답안(01\_플랫폼명-문제번호-풀이방식.js)
5. 문제 답안 설명과 회고(01\_플랫폼명-문제번호-풀이방식.md)
6. 그날 일기를 추가 (../daily-report/오늘날짜.md)

6번은 5번의 내용에 날짜만 추가한 형식이다. 기록은 빠르게 이루어져야 한다. 공부라는 본질에 집중하기.

1,2,3은 자유.
4,5,6이 주.

공부가 끝나면 깃허브 레포 push, 블로그 업로드를 병행한다.

# 문서 역할

- 알고리즘별 문제 파일: 풀이와 회고의 원본입니다.
- `daily-report`: 그날 푼 문제 회고를 빠르게 보기 위한 복사본입니다.
- `docs/templates`: AI가 새 문제 회고 파일을 만들 때 참고하는 템플릿입니다.
- `docs/workflow.md`: 문제 시작과 마무리 자동화 규칙입니다.

자세한 진행 규칙은 `docs/workflow.md`를 따릅니다.
