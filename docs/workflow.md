# Algorithm Study Workflow

별거 없습니다. 시작할 때 문제 파일을 만들고, 끝날 때 데일리 리포트에 원본 회고를 복사합니다.

## 시작할 때

사용자가 새 문제 시작을 요청하면 아래 정보를 확인합니다.

```text
문제번호
문제종류
문제플랫폼
풀이방식
문제제목
```

그다음:

1. `docs/templates/problem.md`를 보고 새 문제 회고 파일을 만듭니다.
2. 같은 이름의 JS 풀이 파일을 최소 형태로 만듭니다.
3. 문제종류 폴더가 없으면 새로 만듭니다.

생성 파일:

```text
{문제종류}/01_{문제플랫폼}-{문제번호}-{풀이방식}.js
{문제종류}/01_{문제플랫폼}-{문제번호}-{풀이방식}.md
```

예시:

```text
bfs/01_boj-2606-union-find.js
bfs/01_boj-2606-union-find.md
```

JS 파일은 템플릿 없이 아래 최소 형태만 둡니다.

```js
function solution(input) {
}
```

## 끝날 때

사용자가 `다했어`처럼 마무리를 요청하면:

1. 방금 만든 문제 md 파일을 읽습니다.
2. 오늘 날짜의 daily report를 확인합니다.
3. 없으면 `daily-report/YYYY-MM-DD.md`로 만듭니다.
4. Day 번호는 기존 daily report 개수를 보고 자동 계산합니다.
5. 오늘 리포트 맨 아래에 문제 md 내용을 그대로 붙입니다.
6. 같은 문제 섹션이 이미 있으면 중복 추가하지 않습니다.
7. `bash scripts/status.sh`를 실행해 `README.md`의 공부 현황(연속 일수·최근 기록)을 갱신합니다.
8. 커밋/푸시는 하지 않습니다.

예시:

```text
다했어
```

## 파일 규칙

- `00_` 파일은 자동화 대상이 아닙니다.
- 문제 번호 접두사는 자동 증가하지 않고 `01_`을 유지합니다.
- 파일명 요소는 전부 케밥 케이스로 씁니다.
- 문제종류는 폴더명입니다.
- 풀이방식은 파일명 마지막 부분입니다.
- 문제 md 파일이 원본이고, daily report는 그날 기록을 빠르게 보기 위한 복사본입니다.

## 기억할 것

- 방금 만든 문제 파일이 불분명하면 마무리 전에 어떤 md 파일을 붙일지 사용자에게 확인합니다.
- daily report에는 문제 md 내용을 수정하지 않고 그대로 복사합니다.
- README나 daily report 갱신만으로 커밋하거나 푸시하지 않습니다.
- 공부 현황은 `scripts/status.sh`가 `README.md`의 `<!-- STATUS:START --> ~ END -->` 구간을 자동 갱신합니다. 이 구간은 손으로 고치지 않습니다.
