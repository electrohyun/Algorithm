## 씨름했던 지점 (JS 손번역)

| 문제         | 함정                                             | 되짚을 포인트                                           |
| ------------ | ------------------------------------------------ | ------------------------------------------------------- |
| 카펫         | `for...in` 으로 배열 순회                        | `in` = 인덱스(문자열), `of` = 값. 배열 돌 땐 `of`       |
| 카펫         | 세로를 brown 하나로 구하려 함                    | brown·yellow **둘 다** 필요. 후보 (가로,세로)마다 검사  |
| 카펫         | 가로≥세로인데 시작을 `squareSize/2` 로           | x의 하한은 절반이 아니라 √n. 세로를 1부터 도는 게 깔끔  |
| 카펫         | 정답으로 안쪽 `[x-2, y-2]` 를 반환               | 반환은 카펫 전체 `[x, y]`. 안쪽 크기는 판정용           |
| 큰 수 만들기 | "남길 조합을 다 만들자" (조합 폭발, n 최대 백만) | 만드는 게 아니라 **버리는** 문제. 한 번 순회로 O(n)     |
| 큰 수 만들기 | 비교를 `number[i]` vs `number[i+1]` 로           | 입력 이웃이 아니라 **스택 맨 뒤 vs 지금 숫자**          |
| 큰 수 만들기 | 하나만 pop 하면 되는 줄                          | 예산·조건 되는 만큼 여러 개 → `if` 아니라 **`while`**   |
| 큰 수 만들기 | 같은 값도 버릴 뻔 (`<=`)                         | `맨뒤 < 지금` **엄격 비교**. 같으면(=) 안 버림          |
| 큰 수 만들기 | 예산 k 남는 경우 처리 누락                       | 내림차순이면 스캔 중 안 버려짐 → 끝에서 뒤 k개 잘라내기 |

## 문제

```js
// ── p1) 큰 수 만들기 (42883) ──
// 그리디 + 스택(모노토닉). 왼쪽부터 한 글자씩, "지금 게 맨 뒤보다 크고 예산(k) 남으면"
// 맨 뒤를 버린다 → 큰 숫자를 앞으로. 다 돌고 k 남으면(내림차순) 뒤에서 잘라냄.
function p1(number, k) {
  const stack = [];

  for (const digit of number) {
    // 맨 뒤가 지금보다 작고, 아직 버릴 예산 있으면 → 계속 버림 (if 아니라 while!)
    while (stack.length > 0 && stack[stack.length - 1] < digit && k > 0) {
      stack.pop();
      k -= 1;
    }
    stack.push(digit); // 안 버려진 것 포함, 지금 숫자는 매번 넣기
  }

  // k가 남았다 = 스캔 중 못 버림(내림차순) → 뒤에서 k개 잘라내기
  return stack.slice(0, stack.length - k).join("");
}

// ── p2) 카펫 (42842) ──
// 전체 칸 = brown + yellow = 가로 × 세로. 약수쌍이 후보 (가로,세로).
// 안쪽 노란 직사각형 (x-2)*(y-2) 가 yellow 와 맞는 쌍이 정답.
function solution(brown, yellow) {
  const squareSize = brown + yellow;

  const xyPair = [];

  for (let x = 1; x <= squareSize; x++) {
    const y = squareSize / x;
    const isTooTall = y > x; // 세로가 가로보다 길면 후보 아님

    if (isTooTall) continue;

    const isDivisor = squareSize % x === 0;

    if (isDivisor) xyPair.push([x, y]);
  }

  for (const [x, y] of xyPair) {
    const yellowWidth = x - 2;
    const yellowHeight = y - 2;

    const isAnswer = yellowWidth * yellowHeight === yellow;

    if (isAnswer) return [x, y];
  }

  return -1;
}
```

## 회고

오늘의 발견:

- 브루트 포스 문제는 그래도 접근이 된다. 알고리즘 재활이 필요하다.

잘한 점:

- const 상수를 잘 배치해서 풀이해줬다. 보기 좋은 것에만 치중하는 게 아니라, 외부에서 보기에도 깔끔한 게 잘 푸는 방법은 아닐까 하는 생각이 든다.

불확실한 점:

- 큰 수 만들기는 꼭 일주일 뒤에 다시 풀러 올 것이다.

내일부터:

- BFS/DFS 이전의 유형은 그래도 잘 풀거라고 생각했는데! 아니었다.
- 알고리즘 재활훈련 하자.
